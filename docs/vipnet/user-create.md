# Заведение пользователя через ViPNet Administrator 4

> Инструкция основана на типовом процессе ViPNet Administrator 4: добавление узла, выпуск сертификатов и выдача данных пользователю.

## 1. Подготовка

- Убедитесь, что у вас есть права администратора УКЦ.
- Заранее узнайте ФИО, подразделение и желаемый идентификатор узла/пользователя.
- Согласуйте, нужна ли почта/телефон в карточке для рассылки ключей.

## 2. Создание узла (пользователя)

1. Откройте **ViPNet Administrator** → вкладка **Узлы** → нажмите **Создать узел**.
2. В мастере выберите тип узла: **Пользователь (Client)**.
3. Заполните поля:
   - **Имя узла / пользователя** (ФИО или уникальное имя);
   - **Организация/подразделение** (по внутреннему справочнику);
   - **Email/телефон** (если используется рассылка данных).
4. Нажмите **Далее** и подтвердите создание.

## 3. Выпуск ключей и сертификатов

1. В панели действий выберите **Выпустить сертификаты** для созданного узла.
2. Задайте **пароль контейнера** для ключей пользователя (передайте безопасным способом).
3. Запустите выпуск и дождитесь завершения операции.

## 4. Формирование дистрибутива/ключевого носителя

1. В карточке узла откройте **Дистрибутив / ключи** → **Создать пакет**.
2. Выберите формат выдачи:
   - Файл настроек/ключей (например, `*.vne` / `*.vpn`),
   - или запись на токен/смарт-карту (если используется аппаратный носитель).
3. Установите опции защиты (пароль на архив/файл, срок действия ссылки и т.п.).
4. Сохраните пакет и проверьте его наличие в каталоге экспорта.

## 5. Выдача пользователю

- Передайте файл/носитель пользователю через согласованный защищённый канал (лично, по защищённой почте или через службу доставки токенов).
- Передайте пароль к контейнеру отдельно (не тем же каналом).

## 6. Проверка подключения

1. После установки пакета на рабочей станции пользователя убедитесь, что клиент вышел в сеть ViPNet и проходит аутентификацию.
2. В **ViPNet Monitor/Administrator** проверьте статус узла: наличие соединения и актуальность сертификата.

## 7. Мини-чек-лист

- [ ] Узел создан и атрибуты заполнены.
- [ ] Сертификаты выпущены, пароль контейнера зафиксирован.
- [ ] Пакет ключей/настроек сгенерирован и сохранён.
- [ ] Пакет и пароль переданы разными каналами.
- [ ] Пользователь подтвердил успешное подключение.

## 8. Где править

- При смене данных пользователя (ФИО, подразделение, контакты) обновите карточку узла и при необходимости перевыпустите сертификаты.
- При компрометации ключей — отзыв сертификата и выпуск нового пакета с новым паролем.
